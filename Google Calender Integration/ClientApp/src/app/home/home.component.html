Startup steps

  1. Create a new project at https://console.cloud.google.com/projectcreate?previousPage=%2Fapis%2Fcredentials%2Fconsent%3Fproject%3Dpeppy-ridge-384306&organizationId=0
  2. Enable services for Google Calendar and Task: https://console.cloud.google.com/welcome?project=peppy-ridge-384306
  3. Setup OAuth screen: https://console.cloud.google.com/apis/credentials/consent?project=peppy-ridge-384306. Ensure to add following scopes:
    • .../auth/calendar
    • .../auth/calendar.events
    • .../auth/tasks
    • .../auth/tasks.readonly
  4. Create OAuth 2.0 Client ID: https://console.cloud.google.com/apis/credentials?project=peppy-ridge-384306. Ensure to add Authorized redirect URIs

OAuth2:

  • https://developers.google.com/identity/protocols/oauth2
  • https://developers.google.com/api-client-library/dotnet/guide/aaa_oauth

Task:

Note: Currently the due date only records date information; the time portion of the timestamp is discarded when setting the due date. It isn't possible to read or write the time that a task is due via the API. https://stackoverflow.com/questions/66744979/google-tasks-doesnt-set-time-only-date
